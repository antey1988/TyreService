Просмотрев путь: ru.tyreservice.aggregator.domain вы можете наблюдать 2 папки, 
в одной из них (entity) описание всех нужных нам на данном этапе разработки моделей,
в другой (enums) описание всех State-ов таких как тип автомобиля, статус заказа.
 Структура БД представлена картинкой DB.jpg.
 Для запуска локального Бэка, а также запуска локальной БД,
 необходимо проделать следующие действия:
 1) Утановить PostgreSQL 12 и при уствноке изменить только имя пользователя и пароль, указав свои данные регистарции,
 далее создать базу данных желательно с именем "market"
 (если решите использовать другое то нужно будет изменить строку о которой пойдет речь дальше)
 2) Установить IntelliJ IDEA или другую среду разработки
 3) Запустить среду разработки и открыть в ней папку проекта с названием "TyreServiceBack"
 4) Пройти по пути src/main/resources/application.properties
 5) В данном файле изменить первые три строки на нужные вам данные:
 Поля взятые в кавычки показывают что вам нужно изменить
 spring.datasource.url=jdbc:postgresql://localhost:5432/"market" (название вашей таблицы из 1*)
 spring.datasource.password="password" (пароль от вашего аккаунта Postgres из 1*)
 spring.datasource.username="postgres" (имя пользователя вашего аккаунта из 1*)
 6) заупустить проект кнопкой Play вверху среды разработки
 7) старндарный url для бэка будет http://localhost:8080/(далее уже будут указываться ссылки на те или иные страницы)
 Например:
 http://localhost:8080/signUp будет отправлять на этот URL JSON  регистрацонными данными Парнера
 http://localhost:8080/list будет получать json со списком всех Партнеров с необходимыми данными
 Все ссылки и какую функцию они выполняют будем описывать в файле README_CONTROLLERS.md.
